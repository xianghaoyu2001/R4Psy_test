# 对 Christensen 等人(2018)研究结果的计算可复现性检验方案

## 所选文献

Christensen, A. P., Kenett, Y. N., Cotter, K. N., Beaty, R. E., & Silvia, P. J. (2018). Remotely Close Associations: Openness to Experience and Semantic Memory Structure. *European Journal of Personality*, *32*(4), 463–479.

- DOI: 10.1002/per.2157
- 数据来源: https://osf.io/craky/

## 文献简介

这篇文章研究的是人格特质"开放性"(Openness to Experience)与语义记忆网络结构之间的关系。作者收集了 516 名被试的语义流畅性任务数据（让被试在规定时间内尽量多地列举动物），然后用了语义网络分析的方法，把每个被试的反应构建成网络，比较高开放性组和低开放性组在网络结构上的差异。

主要发现是：高开放性个体的语义网络更松散，包含更多独特的、别人没有提到的词汇，网络聚类更少。这跟"开放性高的人思维更发散"的直觉是一致的。

## 研究假设

这篇文章只有一个实验，没有多个子研究。我们需要复现的核心假设有三个：

1. 高开放性组和低开放性组在独特反应数量上有差异（McNemar 检验）
2. 语义流畅性反应总数与开放性分数正相关（Pearson 相关）
3. 高/低开放性组在语义网络结构指标（ASPL、CC、Q、S）上有差异（语义网络分析 + bootstrap）

## 数据集选取

原作者在 OSF 上放了完整的数据和代码：

- 原始数据：`All samples merged and totaled_shared.sav`（SPSS 格式，4.4MB）
- 清洗后数据：`FINAL fluency.csv`、`FINAL open.csv`
- 分析代码：`Remotely Close Associations R Script.R`
- 还附了数据清洗的说明文档、实验材料、论文手稿等

按照指南的要求，我们会从原始 SAV 文件开始，走一遍完整的数据清洗和分析流程，而不是直接用清洗好的 CSV。这样才算是真正的计算可复现性检验。

样本量 516，不打算增减。

## 数据分析方法

原论文用的是 R 语言，主要依赖两个包：`SemNetCleaner`（语义网络数据清洗）和 `NetworkToolbox`（网络分析和 bootstrap）。统计方法包括：

- McNemar 检验（比较两组独特反应）
- Pearson 相关（反应总数与开放性的关系）
- 独立样本 t 检验（两组反应数的差异）
- 语义网络指标计算（ASPL、CC、Q、S 四个指标）
- 部分网络 bootstrap 检验

因为原文用的就是 R，原代码可以直接跑，所以核心工作是把原代码跑通、把每一步的结果和原文对比。

另外打算在复现基础上加点创新：用 Bayes 方法重新做 t 检验和相关分析，这样不依赖 p 值也能评估证据强度，也算是对课程学到的 Bayes 统计的实践。

## 复现流程

### 第一步：环境搭建

- R 4.x + RStudio
- 安装包：`SemNetCleaner`、`NetworkToolbox`、`psych`、`haven`、`BayesFactor`
- SemNetCleaner 可能比较老，如果 CRAN 装不上就从 GitHub 装

### 第二步：数据预处理

1. 用 `haven::read_sav()` 读取原始 SAV 文件
2. 用 `SemNetCleaner::semnetcleaner()` 清洗语义流畅性反应（去停用词、合并单复数、统一拼写等）
3. 用 `autoDeStr()` 去除字符串连接（比如被试把"polarbear"写成一个词）
4. 用 `autoConverge()` 合并语义相同的反应（比如"dog"和"dogs"算同一个）
5. 手动修正：原代码里对两个特殊被试（386号、499号）做了手动补录，我们照做
6. 读取 `FINAL open.csv` 中的开放性分数，用 `psych::alpha()` 算 Cronbach's α
7. 按开放性中位数把被试分成 high/low 两组

### 第三步：描述性统计

把样本的基本信息整理出来，和原文对比：

- 总人数、各分组人数
- 开放性分数的均值、标准差
- Cronbach's α（BFAS Openness、BFAS Intellect、NEO 三个量表）
- 两组合计的总反应数

按指南的表 2 格式做成对比表，算 δ 和评级。

### 第四步：推断性统计（原方法）

逐个跑原文的分析：

1. McNemar 检验 → 比 χ² 值
2. Pearson 相关 → 比 r 值
3. t 检验 → 比 t 值，算 Cohen's d
4. 语义网络分析 → 用 TMFG 方法建网络，用 semnetmeas 算四个网络指标
5. 部分网络 bootstrap（50%/60%/70%/80%/90% 五个水平）→ 比网络稳定性

每一项都和原文报告的数字比，算 PE（百分误差），判定完全一致/次要偏差/主要偏差。

### 第五步：创新方法

用 Bayes 方法重做关键分析：

- 独立样本 t 检验 → `BayesFactor::ttestBF()`，报告 BF₁₀
- 相关分析 → `BayesFactor::correlationBF()`，报告 BF₁₀

比较 Bayes 推论和原文 NHST 推论是否一致。

### 第六步：汇总评估

把前面所有结果汇总：

- 结果可复现性表（多少完全一致、多少有偏差）
- 推论一致性表（几个推论一致、几个不一致）
- (不)可复现原因分析（按指南表 10 逐项检查）

## 潜在问题

1. SemNetCleaner 是 2018 年的包，可能在新版 R 上有兼容问题，需要提前测试
2. 原代码里有些硬编码的索引（比如开放性分数在第 10-19 列），需要对照实际数据确认
3. 语义网络 bootstrap 比较吃算力（1000 次迭代 × 5 个水平），跑起来可能要一点时间
4. 原代码最后导出了 Cytoscape 格式做可视化，复现时可以跳过或者用 R 画
